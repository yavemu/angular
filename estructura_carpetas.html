Estructura carpetas de un proyecto Angular

e2e: Carpeta que se usa principalmente para realizar pruebas unitarias y automatizadas.

node_modules: Carpeta donde se almacenan todos los paquetes que se pueden usar en un ambiente de desarrollo.
Esta carpeta no se requiere salvar, al momento de descargar un proyecto se debe ejecutar el comando npm install para recuperar y volver a tener los paquetes en ambiente desarrollo en base al archivo package.json

src: La carpeta Source, es donde se encuentra todo nuestro proyecto de angular, es donde crearemos casi todo nuestro codigo.

.editorconfig: Configuraciones para el editor (No es muy relevante)

.gitignore: Archivo donde se indican que archivos se deben ignorar al realizar copia de respaldo en git.

angular.json: Archivo donde se le indica a angular como es nuestra aplicacion y como funciona. (No se modifica constantemente por nosotros)

package-lock.json: Archivo que se genera automaticamente y en el cual su contenido le indica a nuestra aplicacion como se creo el archivo package.json (No se toca manualmente)

package.json: Archivo que se genera automaticamente, donde se encuentran todas las diferentes dependencias que utilizamos en los diferentes entornos de ejecuci√≥n

tsconfig.json: Archivo que le indica a TypeScript como trabajar, con que estandar de JS.

tslint.json: Es un archivo que nos fuerza a escribir un codigo mas limpio de TypeScript
